## 📊 1. 대시보드 (`/admin_panel/`)

### 핵심 기능

#### 통계 카드 (5개)
1. **신규 가입자 수**: 오늘 가입한 사용자 수 (00:00:00 ~ 23:59:59)
2. **가입된 의사 수**: 검증 완료된 의사 수
3. **총 병원 수**: 등록된 모든 병원 수
4. **미처리 1:1 문의**: 답변이 없는 문의 수
5. **의사 승인 대기**: 검증되지 않은 의사 수

#### 추가 통계
- **총 가입한 의사 수**: 검증 여부와 관계없이 전체 의사 수
- **7일 방문자 수**: 최근 7일간 일일 방문자 수 합계
- **오늘 가입한 회원 (웹/모바일 구분)**: 
  - 웹: 일반 회원가입 (`provider='local'`)
  - 모바일: 소셜 로그인 (`provider__in=['kakao', 'naver']`)

#### 방문자 그래프
- **Chart.js 막대 그래프**: 최근 7일간 일일 방문자 수 시각화
- **데이터 소스**: `DailyVisit` 모델
- **자동 데이터 보완**: 데이터가 없는 날짜는 0으로 표시

#### 기술적 특징
```python
# 오늘 가입한 사용자 수 (정확한 시간 범위)
today_start = timezone.now().replace(hour=0, minute=0, second=0, microsecond=0)
today_end = timezone.now().replace(hour=23, minute=59, second=59, microsecond=999999)
new_users_count = Users.objects.filter(
    created_at__gte=today_start,
    created_at__lte=today_end,
    withdrawal='0'
).count()

# 7일간 방문자 그래프 데이터 생성
visitor_chart_data = {'labels': [], 'values': []}
for i in range(6, -1, -1):  # 6일 전부터 오늘까지
    date = today - timedelta(days=i)
    try:
        daily_visit = DailyVisit.objects.get(visit_date=date)
        count = daily_visit.visit_count
    except DailyVisit.DoesNotExist:
        count = 0
    visitor_chart_data['labels'].append(date.strftime('%m/%d'))
    visitor_chart_data['values'].append(count)
```

## 👥 2. 사용자 관리 (`/admin_panel/user_list/`)

### 핵심 기능

#### 사용자 목록 조회
- **필터링**: 일반 사용자만 (`role='PATIENT'`), 탈퇴하지 않은 사용자 (`withdrawal='0'`)

  - 이메일 (`email`)
  - 전화번호 (`phone`)
- **정렬 기능**: 
  - 사용자 ID, 사용자명, 이름, 이메일, 전화번호, 성별, 주민번호, 주소, 가입일
  - 오름차순/내림차순 선택
- **페이지네이션**: 5개씩 표시
- **신규 가입자 표시**: 오늘 가입한 사용자 뱃지 표시

#### 사용자 상세 정보 (AJAX)
- **클릭 시 상세 정보 표시**: 페이지 전환 없이 AJAX로 상세 정보 로드
- **표시 정보**:
  - 기본 정보 (이름, 이메일, 전화번호, 주소)
  - 생년월일 (주민번호에서 변환)
  - 즐겨찾기 병원 목록
  - 가입일

#### 데이터 가공
- **주민번호 → 생년월일 변환**: `yyyy년mm월dd일` 형식
- **주소 처리**: 
  - 목록: 동까지만 표시
  - 상세: `(우편번호) 메인주소 상세주소` 형식
- **신규 가입자 확인**: 오늘 날짜 범위 내 가입자 표시

#### 기술적 특징
```python
# 주민번호를 생년월일로 변환
reg_no = user.resident_reg_no.replace('-', '')
yy = reg_no[0:2]
mm = reg_no[2:4]
dd = reg_no[4:6]
gender_code = reg_no[6]

# 주민번호 7번째 자리로 연도 판단
if gender_code in ['1', '2', '5', '6']:
    yyyy = f'19{yy}'  # 1900년대
elif gender_code in ['3', '4', '7', '8']:
    yyyy = f'20{yy}'  # 2000년대

birth_date = f'{yyyy}년{mm}월{dd}일'

# 신규 가입자 확인
today_start = timezone.now().replace(hour=0, minute=0, second=0, microsecond=0)
today_end = timezone.now().replace(hour=23, minute=59, second=59, microsecond=999999)
is_new_user = user.created_at >= today_start and user.created_at <= today_end
```

#### 더미 데이터 관리

- **삭제**: 더미 사용자 데이터 일괄 삭제
- **자동 번호 할당**: 기존 더미 사용자와 중복되지 않도록 번호 자동 할당

- **페이지네이션**: 5개씩 표시

#### 면허번호 검증
- **자동 검증**: 면허번호 뒷자리와 주민번호 뒷자리 비교
- **검증 결과 표시**: 유효/무효 뱃지 표시

- **일괄 승인**: 선택한 의사들을 일괄 승인 (`verified=True`)

  - 면허번호 (검증 결과 포함)

  - 가입일

## 🔧 공통 기능 및 기술

### 1. AJAX 요청 처리

```python
# AJAX 요청 감지
if request.headers.get('X-Requested-With') == 'XMLHttpRequest':
    # 부분 템플릿 렌더링
    detail_html = render_to_string('admin_panel/user_list_detail.html', context, request=request)
    return JsonResponse({'detail_html': detail_html})
```

### 2. 페이지네이션
- **공통 함수**: `paginate_queryset(request, queryset, per_page=5)`
- **페이지당 항목 수**: 5개 (사용자, 의사, 병원, 승인 대기, 문의)
- **번호 계산**: 정렬 방향에 따라 번호 계산 로직 변경

```python
def paginate_queryset(request, queryset, per_page=5):
    paginator = Paginator(queryset, per_page)
    page_number = get_request_param(request, 'page', 1)
    try:
        page_number = int(page_number) if page_number else 1
    except (ValueError, TypeError):
        page_number = 1
    page_obj = paginator.get_page(page_number)
    total_count = queryset.count()
    return page_obj, total_count
```

### 3. 성능 최적화
- **select_related**: ForeignKey 관계 미리 로드 (N+1 쿼리 방지)
- **prefetch_related**: ManyToMany 관계 미리 로드
- **annotate**: 집계 함수를 사용한 계산 필드 추가

```python
# select_related 사용 예시
doctors = Doctors.objects.select_related('user', 'hos', 'dep').filter(...)

# annotate 사용 예시
hospitals = Hospital.objects.annotate(
    doctor_count=Count('doctors')
).all()
```

## 🎯 핵심 기술 포인트

### 1. 복잡한 쿼리 처리
- **Case/When**: 조건부 정렬 및 계산
- **annotate**: 집계 함수를 사용한 계산 필드
- **F 객체**: 필드 참조를 통한 계산

### 2. AJAX 통신
- **부분 렌더링**: 페이지 전환 없이 데이터 업데이트
- **에러 처리**: JSON 응답을 통한 에러 메시지 전달

### 3. 데이터 변환
- **주민번호 → 생년월일**: 주민번호 7번째 자리로 연도 판단
- **날짜 포맷팅**: 다양한 형식 지원 (yyyy년mm월dd일, yyyy.mm.dd)
- **주소 처리**: 우편번호|메인주소|상세주소 형식 파싱

### 4. 사용자 경험
- **상태별 우선 정렬**: 답변 대기, 승인 대기 먼저 표시
- **신규 가입자 표시**: 오늘 가입한 사용자 뱃지
- **면허번호 검증 표시**: 유효/무효 뱃지

## 💡 기술적 도전과 해결

### 1. 상태별 우선 정렬
- **문제**: 답변 대기 문의를 먼저 표시해야 함
- **해결**: Case/When을 사용하여 상태값을 계산하고 정렬

### 2. 면허번호 검증
- **문제**: 면허번호와 주민번호 뒷자리 비교
- **해결**: 문자열 슬라이싱을 통한 뒷자리 추출 및 비교

### 3. AJAX 요청 처리
- **문제**: 페이지 전환 없이 검색/정렬 결과 업데이트
- **해결**: AJAX 요청 감지 후 부분 템플릿만 반환

### 4. 자동 데이터 정리
- **문제**: 오래된 문의 데이터 관리
- **해결**: 30일 이전 데이터 자동 삭제 로직 구현

### 5. 병원 추가 기능
- **문제**: AJAX 기반 병원 추가 및 에러 처리
- **해결**: JSON 응답을 통한 성공/실패 메시지 전달, 상세 에러 정보 제공
