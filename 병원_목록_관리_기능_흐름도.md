# 병원 목록 관리 기능 흐름도

## 개요
관리자 패널의 병원 목록 관리 기능은 병원 정보를 조회, 검색, 정렬, 페이지네이션하고, 새로운 병원을 추가하는 기능을 제공합니다.

## 주요 기능
1. **병원 목록 조회**: 전체 병원 목록을 테이블로 표시
2. **검색 기능**: 병원명, 지역, 전화번호, 병원ID로 검색
3. **정렬 기능**: No., 병원ID, 병원명, 주소, 전화번호, 의사 수, 등록일, 개원일로 정렬
4. **페이지네이션**: 5개 항목씩 페이지 단위로 표시
5. **상세 정보 표시**: 병원 행 클릭 시 상세 정보를 AJAX로 로드하여 표시
6. **병원 추가**: 새로운 병원을 추가하는 기능

## 1. 병원 목록 조회 흐름도

```
[사용자] 
  ↓
[병원 목록 페이지 접속] (GET /admin_panel/hospital_list/)
  ↓
[서버: hospital_list 뷰 함수]
  ↓
[파라미터 추출]
  - search_type: 검색 조건 (name, region, phone, hpid)
  - search_keyword: 검색어
  - hospital_id: 선택된 병원 ID (상세 정보 표시용)
  - sort: 정렬 필드
  - order: 정렬 순서 (asc, desc)
  - page: 페이지 번호
  ↓
[기본 쿼리셋 생성]
  - Hospital.objects.annotate(doctor_count=Count('doctors'))
  - 의사 수 포함하여 조회
  ↓
[정렬 적용]
  - sort 필드에 따라 정렬
  - 기본값: -created_at (최신순)
  ↓
[검색 필터 적용]
  - search_type과 search_keyword에 따라 필터링
  - icontains 사용 (대소문자 구분 없음)
  ↓
[페이지네이션]
  - per_page=5 (5개씩 표시)
  - Paginator 사용
  ↓
[데이터 가공]
  - 각 항목에 번호 부여
  - 주소 첫 번째 쉼표까지만 표시 (address_short)
  - 개원일 포맷팅 (yyyy.mm.dd)
  - 등록일 포맷팅 (yyyy.mm.dd)
  ↓
[선택된 병원 정보 처리]
  - hospital_id가 있으면 해당 병원 정보 조회
  - 상세 정보 템플릿용 데이터 준비
  ↓
[통계 정보 계산]
  - total_hospitals_count: 전체 병원 수
  - search_result_count: 검색 결과 수
  ↓
[템플릿 렌더링]
  - hospital_list.html 렌더링
  - 컨텍스트 전달:
    * page_obj: 페이지네이션 객체
    * hospitals_with_number: 번호가 포함된 병원 목록
    * search_type, search_keyword: 검색 조건
    * selected_hospital: 선택된 병원 정보
    * total_hospitals_count, search_result_count: 통계 정보
    * sort_field, sort_order: 정렬 정보
  ↓
[클라이언트: HTML 응답]
  - 병원 목록 테이블 표시
  - 검색 폼 표시
  - 페이지네이션 표시
  - 선택된 병원이 있으면 상세 정보 표시
```

## 2. 검색 기능 흐름도

```
[사용자: 검색 폼 입력]
  - search_type 선택 (병원명, 지역, 전화번호, 병원ID)
  - search_keyword 입력
  - 검색 버튼 클릭
  ↓
[POST 요청 전송]
  - method: POST
  - action: /admin_panel/hospital_list/
  - 데이터: search_type, search_keyword, csrf_token
  ↓
[서버: hospital_list 뷰 함수]
  ↓
[검색 필터 적용]
  if search_type == 'name':
    hospitals.filter(name__icontains=search_keyword)
  elif search_type == 'region':
    hospitals.filter(address__icontains=search_keyword)
  elif search_type == 'phone':
    hospitals.filter(tel__icontains=search_keyword)
  elif search_type == 'hpid':
    hospitals.filter(hos_name__icontains=search_keyword)
  ↓
[정렬 및 페이지네이션 적용]
  ↓
[검색 결과 렌더링]
  - 검색 조건과 검색어 유지
  - 검색 결과 수 표시
  ↓
[클라이언트: 검색 결과 표시]
```

```
[페이지네이션 및 렌더링]
  ↓
[클라이언트: 정렬된 목록 표시]
```

```
[사용자: 병원 검색 모달에서 병원명 입력]
  ↓
[JavaScript: hospital_list.js]
  - debouncedFetchHospitals 함수 호출 (입력 후 300ms 대기)
  ↓
[API 요청]
  - 공공데이터포털 병원 정보 API 호출
  - 검색어로 병원 목록 조회
  ↓
[검색 결과 표시]
  - 검색 결과 목록 표시
  - 병원 선택 가능
  ↓
[사용자: 검색 결과에서 병원 선택]
  ↓
[폼 자동 입력]
  - 병원명, 주소, 전화번호, 개원일 등 자동 입력
  - 위도/경도, 시도/시군구 정보도 함께 저장
  - 검색 모달 닫기
```

### 6.3. 병원 추가 폼 제출

```
[사용자: "추가" 버튼 클릭]
  ↓
[JavaScript: hospital_list.js]
  - 폼 제출 이벤트 처리
  ↓
[클라이언트 사이드 검증]
  - 필수 필드 확인:
    * hospital_name (병원명)
    * hospital_hos_name (병원 계정ID)
    * hospital_hos_password (병원 비밀번호)
  - 필드 길이 검증
  - 에러 필드에 'error' 클래스 추가
  ↓
[POST 요청 전송]
  - method: POST
  - action: /admin_panel/hospital_list/
  - 데이터:
    * action: 'add_hospital'
    * hospital_name, hospital_hos_name, hospital_hos_password
    * hospital_address, hospital_tel, hospital_estb_date
    * hospital_lat, hospital_lng, hospital_dr_total
    * hospital_sggu, hospital_sido
    * csrf_token
  - X-Requested-With: XMLHttpRequest 헤더 포함
  ↓
[서버: hospital_list 뷰 함수]
  ↓
[POST 요청 처리]
  - action == 'add_hospital' 확인
  - is_ajax 확인
  ↓
[필수 필드 검증]
  - hospital_name, hospital_hos_name, hospital_hos_password 필수
  - 없으면 에러 응답
  ↓
[필드 길이 검증]
  - hospital_name: 100자 이하
  - hospital_hos_name: 50자 이하
  - hospital_address: 255자 이하
  - hospital_tel: 50자 이하
  ↓
[중복 확인]
  - Hospital.objects.filter(hos_name=hospital_hos_name).exists()
  - 중복이면 에러 응답
  ↓
[hpid 자동 생성]
  - UUID 기반 고유 ID 생성 (36자)
  ↓
[추가 정보 파싱]
  - lat, lng: float 변환
  - dr_total: int 변환
  - sggu, sido: 문자열 (빈 문자열이면 None)
  ↓
[병원 생성]
  - Hospital.objects.create(...)
  ↓
[저장 확인]
  - 저장된 병원 조회하여 확인
  ↓
[성공 응답]
  - AJAX: {'success': True, 'message': '...', 'hospital_id': ...}
  - 일반: redirect('hospital_list')
  ↓
[클라이언트: 성공 처리]
  - 성공 메시지 표시
  - 모달 닫기
  - 목록 새로고침 또는 새로 추가된 병원으로 이동
```

### 6.4. 에러 처리

```
[서버: 에러 발생]
  ↓

  ↓
[클라이언트: 에러 처리]

  - 에러 필드에 'error' 클래스 추가
```

## 8. 데이터베이스 모델 관계

```
Hospital (병원)
  ├── hos_id (PK)
  ├── hpid (고유 ID, UUID 기반)
  ├── hos_name (병원 계정ID, Unique)
  ├── hos_password (병원 비밀번호)
  ├── name (병원명)
  ├── address (주소)
  ├── tel (전화번호)
  ├── estb_date (개원일)
  ├── lat, lng (위도/경도)
  ├── dr_total (의사 수, API에서 가져온 값)
  ├── sggu (시군구)
  ├── sido (시도)
  └── created_at (등록일시)
        │
        └── Doctors (의사) - ForeignKey
              └── doctor_count (의사 수, Count로 계산)
```

## 10. 주요 함수 및 메서드

### 서버 사이드 (views.py)

- `hospital_list(request)`: 병원 목록 조회, 검색, 정렬, 페이지네이션, 병원 추가 처리
- `paginate_queryset(request, queryset, per_page=5)`: 쿼리셋 페이지네이션 처리
- `get_request_param(request, param_name, default='')`: GET/POST 파라미터 추출

### 클라이언트 사이드 (JavaScript)

- `selectHospital(event, hospitalId)`: 병원 선택 함수
- `openAddHospitalModal()`: 병원 추가 모달 열기
- `closeAddHospitalModal()`: 병원 추가 모달 닫기
- `debouncedFetchHospitals`: 병원 검색 API 호출 (디바운싱)
- `attachTableRowListeners()`: 테이블 행 클릭 이벤트 연결 (admin_common.js)
- `attachSortListeners()`: 정렬 링크 이벤트 연결 (admin_common.js)
- `handlePaginationAjax()`: 페이지네이션 AJAX 처리 (admin_common.js)

## 11. 검증 규칙

### 병원 추가 시 검증

1. **필수 필드**
   - 병원명 (hospital_name)
   - 병원 계정ID (hospital_hos_name)
   - 병원 비밀번호 (hospital_hos_password)

2. **필드 길이 제한**
   - 병원명: 최대 100자
   - 병원 계정ID: 최대 50자
   - 주소: 최대 255자
   - 전화번호: 최대 50자

3. **중복 확인**
   - 병원 계정ID (hos_name) 중복 확인

4. **자동 생성**
   - hpid: UUID 기반 고유 ID (36자)

## 12. 에러 처리

### 서버 사이드 에러 처리

1. **IntegrityError (DB 제약 조건 위반)**
   - hpid 중복: "이미 존재하는 병원입니다. (hpid 중복)"
   - hos_name 중복: "이미 존재하는 병원 계정ID입니다."

2. **ValidationError (입력값 검증 오류)**
   - "입력값 검증 오류: {error_message}"

3. **ValueError (값 오류)**
   - "값 오류: {error_message}"

4. **기타 Exception**
   - "병원 추가 중 오류가 발생했습니다: {error_message}"

### 클라이언트 사이드 에러 처리

1. **필수 필드 누락**
   - 에러 필드에 'error' 클래스 추가
   - 에러 메시지 표시

2. **서버 에러 응답**
   - 모달 내부에 에러 메시지 표시
   - 스크롤을 에러 메시지로 이동

## 참고사항

- 모든 AJAX 요청은 `X-Requested-With: XMLHttpRequest` 헤더를 포함해야 합니다.
- 페이지네이션 후에는 이벤트 리스너를 재연결해야 합니다 (`reattachTableRowListeners`).
- 검색, 정렬, 페이지네이션 파라미터는 URL에 유지되어야 합니다.
- 병원 추가 시 API에서 가져온 추가 정보(위도/경도, 시도/시군구 등)도 함께 저장됩니다.
