# 관리자 1:1 문의창 기능 흐름도

## 개요
관리자가 사용자들의 1:1 문의를 조회하고 답변하는 기능입니다.

## 주요 기능
1. **문의 목록 조회**: 전체 문의 목록을 테이블로 표시
2. **정렬 기능**: No., 이름, 이메일, 제목, 일자, 상태로 정렬
3. **페이지네이션**: 5개 항목씩 표시
4. **일괄 삭제**: 선택한 문의를 일괄 삭제 (답변 완료된 문의만)
5. **문의 상세 조회**: 문의 상세 정보 확인
6. **답변 작성/수정**: 문의에 대한 답변 작성 및 수정
7. **자동 삭제**: 30일 이전 문의 데이터 자동 삭제

## 1. 문의 목록 조회 (`qna_list`)

### URL
- `/admin_panel/qna_list/`

### 뷰 함수
- `apps/admin_panel/views.py` → `qna_list()`

### 흐름도

```
[관리자 로그인]
    ↓
[1:1문의관리 탭 클릭]
    ↓
[GET /admin_panel/qna_list/]
    ↓
[관리자 권한 체크]
    ├─ 권한 없음 → 리다이렉트(/)
    └─ 권한 있음 → 계속
    ↓
[30일 이전 데이터 자동 삭제]
    ├─ timezone.now() - timedelta(days=30) 계산
    ├─ Qna.objects.filter(created_at__lt=thirty_days_ago).delete()
    └─ 삭제된 개수 로그 출력 (선택사항)
    ↓
[POST 요청 처리 (삭제)]
    ├─ action == 'delete' 확인
    ├─ qna_ids 파라미터 추출
    ├─ 답변 완료된 문의만 삭제 (reply__isnull=False)
    └─ 리다이렉트(/admin_panel/qna_list/)
    ↓
[정렬 파라미터 추출]
    ├─ sort_field: 정렬 필드 (qna_id, name, email, title, status, created_at)
    └─ sort_order: 정렬 방향 (asc, desc)
    ↓
[쿼리셋 생성]
    ├─ Qna.objects.select_related('user').all()
    └─ 정렬 적용
        ├─ status 정렬: 답변 대기(0) → 답변 완료(1) 우선
        └─ 기본 정렬: 답변 대기 우선, 그 다음 최신순
    ↓
[페이지네이션]
    ├─ per_page=5
    ├─ page_number 파라미터 추출
    └─ Paginator로 페이지 객체 생성
    ↓
[번호 계산]
    ├─ start_index = (page_number - 1) * 5 + 1
    └─ 각 항목에 번호 부여
    ↓
[템플릿 렌더링]
    └─ admin_panel/qna_list.html
```

```

    ↓
```

```
    ↓
[관리자 권한 체크]
    ├─ 권한 없음 → 리다이렉트(/)
```

## 4. 주요 기능 상세

### 4.1. 정렬 기능

#### 정렬 필드
- `qna_id`: 문의 번호
- `name`: 작성자 이름 (user__name)
- `email`: 작성자 이메일 (user__email)
- `title`: 문의 제목
- `status`: 답변 상태 (reply 필드 기준)
- `created_at`: 문의 작성일시

#### 정렬 로직
```python
# 상태 정렬 (특별 처리)
if sort_field == 'status':
    # 답변 대기(0) → 답변 완료(1) 우선
    qnas = qnas.annotate(
        reply_status=Case(
            When(reply__isnull=True, then=Value(0)),
            When(reply='', then=Value(0)),
            default=Value(1),
            output_field=IntegerField(),
        )
    ).order_by(f'{order_prefix}reply_status', '-created_at')

# 기본 정렬
else:
    order_prefix = '-' if sort_order == 'desc' else ''
    qnas = qnas.order_by(f'{order_prefix}{sort_fields[sort_field]}')
```

### 4.2. 페이지네이션

- **페이지당 항목 수**: 5개
- **번호 계산**: 페이지네이션 고려하여 순번 계산
- **AJAX 지원**: 페이지 이동 시 페이지 새로고침 없이 목록 업데이트

### 4.3. 삭제 기능

#### 삭제 조건
- **답변 완료된 문의만 삭제 가능**: `reply__isnull=False`
- **대기 상태 문의는 삭제 불가**: 체크박스 disabled 처리

#### 삭제 처리
```python
# POST 요청 처리
if action == 'delete' and qna_ids_str:
    qna_ids = [int(id.strip()) for id in qna_ids_str.split(',') if id.strip()]
    if qna_ids:
        # 답변 완료된 문의만 삭제
        Qna.objects.filter(qna_id__in=qna_ids, reply__isnull=False).delete()
```

### 4.4. 자동 삭제 기능

- **30일 이전 데이터 자동 삭제**: 매 요청마다 실행
- **삭제 조건**: `created_at < (현재 시간 - 30일)`
- **목적**: 데이터베이스 용량 관리 및 개인정보 보호

```python
thirty_days_ago = timezone.now() - timedelta(days=30)
deleted_count, _ = Qna.objects.filter(created_at__lt=thirty_days_ago).delete()
```

### 4.5. 상태 표시

- **답변 완료**: 초록색 배지 (`status-completed`)
- **대기**: 주황색 배지 (`status-waiting`)

## 6. 보안 고려사항

1. **관리자 권한 체크**: 모든 뷰에서 `user_role == 'ADMIN'` 확인
2. **CSRF 토큰**: POST 요청 시 CSRF 토큰 필수 사용
3. **입력 검증**: 답변 내용 빈 문자열 확인
4. **삭제 제한**: 대기 상태 문의는 삭제 불가
5. **개인정보 보호**: 연락처 마스킹 처리

## 7. 사용자 경험(UX) 개선 사항

1. **시각적 피드백**: 선택된 행에 `checkbox-checked` 클래스 추가
2. **상태 배지**: 답변 완료/대기 상태를 색상으로 구분
3. **확인 대화상자**: 삭제, 답변 완료/취소 시 확인 메시지 표시
4. **AJAX 페이지네이션**: 페이지 이동 시 페이지 새로고침 없이 목록 업데이트
5. **정렬 방향 표시**: 현재 정렬 필드에 화살표(↑/↓) 표시
6. **체크박스 제한**: 대기 상태 문의는 체크박스 disabled 처리
