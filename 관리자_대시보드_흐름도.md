# 관리자 대시보드 기능 구현 흐름도

## 전체 흐름 개요

```
사용자 요청 → URL 라우팅 → 권한 체크 → 데이터 수집 → 템플릿 렌더링 → 클라이언트 렌더링
```

---

## 1. URL 라우팅 단계

### 1.1 URL 패턴 매칭
```
사용자 접속: /admin_panel/
    ↓
config/urls.py
    ↓
apps/admin_panel/urls.py
    ↓
path('', views.dashboard, name='admin_dashboard')
```

**관련 파일:**
- `config/urls.py`: 메인 URL 설정
- `apps/admin_panel/urls.py`: 관리자 패널 URL 설정

---

## 2. 서버 사이드 처리 (views.py)

### 2.1 권한 체크
```python
def dashboard(request):
    # 세션에서 사용자 역할 확인
    user_role = request.session.get('role', '')
    
    if user_role != 'ADMIN':
        return redirect('/')  # 관리자가 아니면 홈으로 리다이렉트
```

**흐름:**
```
요청 수신
    ↓
세션에서 role 확인
    ↓
role == 'ADMIN'?
    ├─ YES → 다음 단계 진행
    └─ NO  → 홈으로 리다이렉트
```

---

### 2.2 날짜 범위 설정
```python
today = timezone.now().date()
today_start = timezone.now().replace(hour=0, minute=0, second=0, microsecond=0)
today_end = timezone.now().replace(hour=23, minute=59, second=59, microsecond=999999)
week_ago = today - timedelta(days=7)
```

**목적:**
- 오늘 날짜 기준 통계 계산
- 최근 7일간 데이터 조회 범위 설정

---

### 2.3 통계 데이터 수집

#### 2.3.1 신규 가입자 수
```
Users 모델 조회
    ↓
필터 조건:
  - created_at >= today_start
  - created_at <= today_end
  - withdrawal == '0'
    ↓
count() → new_users_count
```

#### 2.3.2 검증 완료된 의사 수
```
Doctors 모델 조회
    ↓
필터 조건:
  - verified == True
    ↓
count() → verified_doctors_count
```

#### 2.3.3 총 병원 수
```
Hospital 모델 조회
    ↓
count() → total_hospitals_count
```

#### 2.3.4 총 의사 수
```
Doctors 모델 조회
    ↓
count() → total_doctors_count
```

#### 2.3.5 7일 방문자 수 합계
```
DailyVisit 모델 조회
    ↓
필터 조건:
  - visit_date >= week_ago
  - visit_date <= today
    ↓
aggregate(Sum('visit_count')) → weekly_visitors_count
```

#### 2.3.6 미처리 1:1 문의 수
```
Qna 모델 조회
    ↓
필터 조건:
  - reply__isnull == True
    ↓
count() → pending_qna_count
```

#### 2.3.7 의사 승인 대기 수
```
Doctors 모델 조회
    ↓
필터 조건:
  - verified == False
    ↓
count() → pending_doctors_count
```

#### 2.3.8 오늘 가입한 회원 (웹/모바일 구분)
```
웹 가입자:
  Users 모델 조회
    ↓
  필터 조건:
    - created_at__date == today
    - provider == 'local'
    - withdrawal == '0'
      ↓
  count() → new_users_web

모바일 가입자:
  Users 모델 조회
    ↓
  필터 조건:
    - created_at__date == today
    - provider__in == ['kakao', 'naver']
    - withdrawal == '0'
      ↓
  count() → new_users_mobile
```

---

### 2.4 7일 방문자 그래프 데이터 생성

```
초기화: visitor_chart_data = {'labels': [], 'values': []}
    ↓
for i in range(6, -1, -1):  # 6일 전부터 오늘까지
    ↓
날짜 계산: date = today - timedelta(days=i)
    ↓
날짜 문자열 변환: date_str = date.strftime('%m/%d')
    ↓
DailyVisit 모델에서 해당 날짜 데이터 조회
    ├─ 존재함 → visit_count 값 사용
    └─ 없음 → count = 0
    ↓
visitor_chart_data['labels'].append(date_str)
visitor_chart_data['values'].append(count)
    ↓
JSON 문자열로 변환: json.dumps(visitor_chart_data)
```

**데이터 구조 예시:**
```json
{
  "labels": ["12/01", "12/02", "12/03", "12/04", "12/05", "12/06", "12/07"],
  "values": [100, 150, 120, 180, 200, 170, 190]
}
```

## 3. 클라이언트 사이드 처리

### 3.1 HTML 템플릿 구조

```
dashboard.html
    ├─ 헤더 ({% include 'core/_header.html' %})
    ├─ 요약 카드 섹션
    │   ├─ 신규 가입자수 카드 ({{ new_users_count }})
    │   ├─ 총 가입한 의사 수 카드 ({{ total_doctors_count }})
    │   ├─ 총 병원 수 카드 ({{ total_hospitals_count }})
    │   ├─ 승인 대기중인 의사 카드 ({{ pending_doctors_count }})
    │   └─ 미처리 1:1 문의 카드 ({{ pending_qna_count }})
    └─ 그래프 섹션
        └─ <canvas id="visitorChart"></canvas>
```

---

### 3.2 데이터 전달 (템플릿 → JavaScript)

```
템플릿에서 JSON 데이터 전달:
<script type="application/json" id="visitor-chart-data">
  {{ visitor_chart_data|safe }}
</script>
```

**데이터 형식:**
- JSON 문자열로 전달
- `|safe` 필터로 HTML 이스케이핑 방지

---

### 3.3 JavaScript 차트 생성 (dashboard.js)

```
DOMContentLoaded 이벤트 발생
    ↓
데이터 요소 조회: document.getElementById('visitor-chart-data')
    ↓
JSON 파싱: JSON.parse(visitorDataElement.textContent)
    ↓
캔버스 요소 조회: document.getElementById('visitorChart')
    ↓
Chart.js로 막대 그래프 생성:
  new Chart(ctx, {
    type: 'bar',
    data: {
      labels: visitorData.labels,
      datasets: [{
        label: '일일 방문자 수',
        data: visitorData.values,
        backgroundColor: 'rgba(0, 102, 204, 0.7)',
        borderColor: 'rgba(0, 102, 204, 1)',
        borderWidth: 1
      }]
    },
    options: {
      responsive: true,
      maintainAspectRatio: false,
      scales: {
        y: {
          beginAtZero: true,
          ticks: { stepSize: 50 }
        }
      }
    }
  })
    ↓
차트 렌더링 완료
```

## 5. 주요 통계 데이터 상세

### 5.1 요약 카드 데이터

| 카드 | 데이터 소스 | 필터 조건 | 표시 값 |
|------|------------|----------|---------|
| 신규 가입자수 | `Users` | `created_at` (오늘), `withdrawal='0'` | `new_users_count` |
| 총 가입한 의사 수 | `Doctors` | 전체 | `total_doctors_count` |
| 총 병원 수 | `Hospital` | 전체 | `total_hospitals_count` |
| 승인 대기중인 의사 | `Doctors` | `verified=False` | `pending_doctors_count` |
| 미처리 1:1 문의 | `Qna` | `reply__isnull=True` | `pending_qna_count` |

### 5.2 그래프 데이터

| 항목 | 설명 | 데이터 소스 |
|------|------|------------|
| 기간 | 최근 7일 | `today - 6일` ~ `today` |
| X축 | 날짜 레이블 | `date.strftime('%m/%d')` |
| Y축 | 일일 방문자 수 | `DailyVisit.visit_count` |

## 7. 주요 함수 및 메서드

### 7.1 서버 사이드 (views.py)

- `dashboard(request)`: 대시보드 메인 뷰 함수
- `timezone.now()`: 현재 시간 가져오기


## 9. 성능 최적화

### 9.1 데이터베이스 쿼리 최적화
- `count()`: 전체 레코드 조회 없이 개수만 계산
- `aggregate(Sum())`: 집계 함수 사용으로 효율적 계산
- 필요한 필드만 조회

### 9.2 클라이언트 사이드 최적화
- Chart.js CDN 사용 (캐싱 활용)
- 반응형 차트로 다양한 화면 크기 지원
- DOMContentLoaded 이벤트로 DOM 준비 후 실행
