# CareBridge 프로젝트 기술 구현 내용

## 📋 프로젝트 개요
- **프로젝트명**: CareBridge (케어브릿지)

## 🔧 주요 기술 구현

### 1. 실시간 응급실 조회 시스템

#### 핵심 기능
- **위치 기반 거리 계산**: Haversine 공식을 사용한 정확한 거리 계산
- **혼잡도 점수 계산**: 다중 가중치 알고리즘
  - 응급실 일반 가용률 45%
  - 소아 가용률 20%
  - 음압 가용률 20%
  - 일반 격리 가용률 10%
  - 분만실 가용 여부 5%
- **응급 유형별 필터링**: 뇌졸중, 교통사고, 심장/흉부, 산모/분만
- **장비 필터링**: CT, MRI, 혈관조영, 인공호흡기, 분만실
- **종합 점수 계산**: 거리 + 혼잡도 + 장비 가용성

### 2. 전자의무기록 (EMR) 시스템

#### 핵심 기능
- **진료기록 작성**: SOAP 형식 (Subjective, Objective, Assessment, Plan)
- **처방 관리**: 약품 검색 API 연동, 복용 기간 관리
- **검사 오더**: 검사 항목 선택, 상태 추적 (Pending → Sampled → Completed)
- **치료 오더**: 치료 항목 선택, 실행/완료 상태 관리
- **환자 검색**: 이름, 주민번호 기반 검색
- **과거 진료기록 조회**: 전체 진료 이력 조회

#### 기술적 특징
- **WebSocket 실시간 업데이트**: Django Channels + Redis
  - 검사/치료 상태 변경 시 실시간 차트 업데이트
  - 병원 그룹별 브로드캐스팅
- **복잡한 쿼리 최적화**: select_related, prefetch_related 사용
- **데이터 무결성**: 예약 충돌 방지, 시간대 관리

```python
# WebSocket 실시간 업데이트
channel_layer = get_channel_layer()
group_name = f'hospital_group_{hos_id}'
async_to_sync(channel_layer.group_send)(
    group_name,
    {"type": "chart_update_event", "message": "lab"}
)
```

#### 주요 모델
- MedicalRecord: 진료기록
- MedicineOrders, MedicineData: 처방전
- LabOrders, LabUpload: 검사 오더 및 결과 파일
- TreatmentProcedures: 치료 오더

### 4. 관리자 패널

#### 핵심 기능
- **대시보드**: 주요 통계 정보 및 그래프
  - 최근 7일간 일일 방문자 수 (Chart.js)
  - 사용자/의사/병원/승인대기/문의 통계
- **사용자 관리**: 목록 조회, 검색, 더미 데이터 생성/삭제
- **의사 관리**: 목록 조회, 검색, 더미 데이터 생성/삭제
- **병원 관리**: 목록 조회, 검색
- **승인 대기**: 병원/의사 가입 승인 처리
- **1:1 문의 관리**: 문의 목록, 상세 조회, 답변 작성

#### 기술적 특징
- **AJAX 기반 페이지네이션**: 페이지 전환 없이 데이터 로드
- **세션 기반 필터링**: 정렬, 검색 조건 유지
- **반응형 디자인**: 모바일, 태블릿, PC 지원

### 5. 데이터 수집 시스템

#### 공공 API 연동
1. **HIRA 병원정보 API**: 병원 기본 정보 수집
   - 시도별 병원 목록 (서울/경기 전체, 기타 지역 100개)
   - 병원명, 주소, 좌표, 전화번호 등
2. **보건복지부 응급실 정보 API**: 응급실 실시간 상태
   - 병상 가용 수, 장비 보유 여부
3. **감염병 통계 API**: 감염병 발생 현황
   - 연령별, 성별, 지역별 통계
4. **약품 정보 API**: 처방전 작성 시 약품 검색
5. **공휴일 정보 API**: 예약 시스템에서 공휴일 처리

#### 배치 스크립트
- `import_hospital_data.py`: 병원 정보 수집
- `fetch_emergency.py`: 응급실 상태 수집
- `fetch_emergency_message.py`: 응급실 메시지 수집
- `import_disease_data.py`: 감염병 데이터 수집
- `fetch_medical_news.py`: 의료 뉴스 수집

## 💡 기술적 도전과 해결

### 1. 응급실 혼잡도 계산
- **문제**: 다양한 병상 타입을 하나의 점수로 통합
- **해결**: 가중치 기반 알고리즘 설계, 응급 유형별 가중치 조정

### 2. 예약 충돌 방지
- **문제**: 동일 시간대 중복 예약 방지
- **해결**: 의사/환자 기준 이중 검사, 트랜잭션 처리

### 3. 실시간 데이터 업데이트
- **문제**: 검사/치료 상태 변경 시 실시간 반영
- **해결**: WebSocket + Redis 채널 레이어 구현

### 4. 대용량 데이터 처리
- **문제**: 공공 API 대량 데이터 수집 및 저장
- **해결**: 배치 처리, 페이지네이션, 에러 핸들링 강화
