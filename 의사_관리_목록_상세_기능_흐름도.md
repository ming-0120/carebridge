# 의사 관리 목록 상세 창 기능 흐름도

## 개요
의사 관리 목록 페이지에서 의사 목록을 조회하고, 특정 의사를 선택하여 상세 정보를 확인하는 기능의 전체 흐름을 설명합니다.

## 주요 기능별 상세 흐름

### 1. 검색 기능 흐름

```
사용자 검색 조건 선택 및 검색어 입력
         │
         ▼
검색 버튼 클릭
         │
         ▼
POST 요청 (/admin_panel/doctor_list/)
         │
         ▼
doctor_list 뷰 함수 실행
         │
         ▼
검색 필터 적용
  - search_type == 'name' → user__name__icontains
  - search_type == 'doctor_id' → user__username__icontains
  - search_type == 'license_no' → license_no__icontains
  - search_type == 'department' → dep__dep_name__icontains
  - search_type == 'hospital' → hos__name__icontains
         │
         ▼
페이지네이션 처리
         │
         ▼
검색 결과 표시
```

### 2. 정렬 기능 흐름

```
테이블 헤더 클릭 (예: 이름, 의사ID, 가입일 등)
         │
         ▼
정렬 파라미터 생성
  - sort: 정렬 필드 (name, doctor_id, created_at 등)
  - order: 정렬 순서 (asc/desc)
         │
         ▼
URL에 파라미터 추가하여 페이지 이동
         │
         ▼
doctor_list 뷰 함수 실행
         │
         ▼
정렬 적용
  - sort_fields 매핑으로 DB 필드명 변환
  - order_by() 적용
         │
         ▼
정렬된 결과 표시
```

### 3. 페이지네이션 기능 흐름

```
페이지 번호 클릭
         │
         ▼
페이지 파라미터 추가
  - page: 페이지 번호
  - 기존 검색/정렬 파라미터 유지
         │
         ▼
AJAX 요청 또는 일반 요청
         │
         ▼
doctor_list 뷰 함수 실행
         │
         ▼
페이지네이션 처리
  - paginate_queryset(request, doctors, per_page=5)
  - 각 항목에 번호 부여
         │
         ▼
해당 페이지 데이터 표시
```

### 4. 상세 정보 표시 흐름 (AJAX)

```
테이블 행 클릭
         │
         ▼
selectDoctor(event, doctorId) 호출
         │
         ▼
selectItem(event, doctorId, 'doctor_id') 호출
         │
         ▼
AJAX 요청 생성
  - URL: 현재 URL + ?doctor_id={doctorId}
  - 헤더: X-Requested-With: XMLHttpRequest
         │
         ▼
서버 AJAX 요청 처리
  - X-Requested-With 확인
  - selected_doctor 조회
  - doctor_list_detail.html 렌더링
  - JsonResponse 반환
         │
         ▼
클라이언트 응답 처리
  - detail_html 추출
  - .doctor-detail-section 업데이트
  - 선택된 행에 'selected' 클래스 추가
  - URL 파라미터 업데이트 (history.pushState)
         │
         ▼
상세 정보 표시
```

### 서버 사이드

1. **뷰 함수**: `apps/admin_panel/views.py`

## 참고사항

- 의사 목록은 탈퇴하지 않은 의사만 표시 (`user__withdrawal='0'`)
- 기본 정렬: 승인 대기 먼저, 그 다음 최신순
- 페이지네이션 후 Python `sorted()` 함수 사용하지 않음 (DB 레벨에서 정렬)
- AJAX 요청과 일반 요청을 구분하여 처리
- URL 파라미터로 상태 관리 (검색, 정렬, 페이지, 선택된 의사)
