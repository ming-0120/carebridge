# CareBridge 1:1 문의 시스템 기술 구현

## 📋 개요
- **기능명**: 1:1 문의 시스템 (QnA)
- **목적**: 사용자와 관리자 간의 1:1 문의 및 답변 기능 제공
- **구현 범위**: 관리자 패널 + 사용자 메인 페이지

---

## 🏗️ 시스템 구조

### 데이터베이스 모델
```python
class Qna(models.Model):
    qna_id = models.AutoField(primary_key=True)
    title = models.CharField(max_length=255)          # 문의 제목
    content = models.TextField()                      # 문의 내용
    reply = models.TextField(null=True, blank=True)  # 답변 내용
    created_at = models.DateTimeField(auto_now_add=True)  # 작성일
    privacy = models.CharField(max_length=10)        # 공개/비공개 설정
    user = models.ForeignKey('Users', on_delete=models.CASCADE)  # 작성자
```

## 👨‍💼 관리자 패널 기능

### 1. 대시보드 통계
- **미처리 문의 수 표시**: 답변이 없는 문의 개수 실시간 표시
- **카드 클릭 시 문의 목록으로 이동**

```python
# 대시보드에서 미처리 문의 수 계산
pending_qna_count = Qna.objects.filter(reply__isnull=True).count()
```

### 2. 문의 목록 조회 (`/admin_panel/qna_list/`)

#### 핵심 기능
- **전체 문의 조회**: 모든 사용자의 문의 내역 조회
- **정렬 기능**: 
  - 문의 ID, 작성자명, 이메일, 제목, 상태, 작성일 기준 정렬
  - 오름차순/내림차순 선택 가능
- **상태별 우선 정렬**: 답변 대기 상태가 먼저 표시
- **페이지네이션**: 5개씩 표시
- **일괄 삭제**: 답변 완료된 문의만 일괄 삭제 가능
- **자동 정리**: 30일 이전 문의 데이터 자동 삭제

#### 기술적 특징
```python
# 상태별 우선 정렬 (답변 대기 먼저)
qnas = qnas.annotate(
    reply_status=Case(
        When(reply__isnull=True, then=Value(0)),  # 대기 = 0
        When(reply='', then=Value(0)),             # 대기 = 0
        default=Value(1),                         # 완료 = 1
        output_field=IntegerField(),
    )
).order_by('reply_status', '-created_at')

# 30일 이전 데이터 자동 삭제
thirty_days_ago = timezone.now() - timedelta(days=30)
deleted_count, _ = Qna.objects.filter(
    created_at__lt=thirty_days_ago
).delete()
```

#### 성능 최적화
- `select_related('user')`: N+1 쿼리 문제 방지
- 페이지네이션으로 대량 데이터 처리

### 3. 문의 상세 조회 및 답변 작성 (`/admin_panel/qna_detail/<qna_id>/`)

#### 핵심 기능
- **문의 상세 정보 표시**: 제목, 내용, 작성자 정보, 작성일
- **답변 작성**: 관리자가 답변 내용 입력 및 저장
- **답변 취소**: 답변 작성 취소 후 목록으로 이동

#### 기술적 특징
```python
# 문의 조회 (사용자 정보 미리 로드)
qna = get_object_or_404(
    Qna.objects.select_related('user'), 
    qna_id=qna_id
)

# 답변 저장
if request.method == 'POST' and action == 'reply':
    reply_content = request.POST.get('reply_content', '').strip()
    if reply_content:
        qna.reply = reply_content
        qna.save()
        return redirect('/admin_panel/qna_list/')
```

### 4. 더미 데이터 관리

#### 더미 데이터 생성
- 테스트용 문의 데이터 자동 생성
- 다양한 문의 유형 포함 (서비스 문의, 예약 문의, 오류 신고 등)
- 일부 문의는 자동으로 답변 포함

#### 더미 데이터 삭제
- 제목이 "더미 문의"로 시작하는 문의만 삭제
- 실제 사용자 문의는 보호

## 👤 사용자 측 기능

### 1. 메인 페이지 연동
- **QnA 카드**: 메인 페이지에서 "문의하기" 버튼 클릭 시 문의 목록으로 이동
- **직접 접근**: `/qna/` URL로 직접 접근 가능

### 2. 문의 목록 조회 (`/qna/`)

#### 핵심 기능
- **본인 문의 조회**: 로그인한 사용자가 작성한 모든 문의 (공개/비공개 모두)
- **공개 문의 조회**: 다른 사용자가 공개로 설정한 문의도 조회 가능
- **검색 기능**: 제목, 내용으로 검색
- **정렬 기능**: 문의 ID, 제목, 작성자, 공개설정, 상태, 작성일 기준 정렬
- **페이지네이션**: 10개씩 표시
- **AJAX 지원**: 페이지 전환 없이 검색/정렬 결과 업데이트

#### 기술적 특징
```python
# 본인 문의 + 공개 문의 조회
qnas = Qna.objects.select_related('user').filter(
    Q(user=user) |  # 본인 문의 (공개/비공개 모두)
    (Q(privacy='PUBLIC') & ~Q(user=user))  # 다른 사람의 공개 문의
).filter(user__withdrawal='0')  # 탈퇴하지 않은 사용자만

# 상태 정렬 (답변 대기 먼저)
qnas = qnas.annotate(
    status_value=Case(
        When(reply__isnull=True, then=Value(0)),  # 대기
        default=Value(1),                         # 완료
        output_field=IntegerField()
    )
).order_by('status_value')
```

#### AJAX 처리
```python
# AJAX 요청인 경우 테이블 부분만 반환
if request.headers.get('X-Requested-With') == 'XMLHttpRequest':
    table_html = render_to_string('qna/partials/qna_table.html', context)
    pagination_html = render_to_string('qna/partials/qna_pagination.html', context)
    return JsonResponse({
        'table_html': table_html,
        'pagination_html': pagination_html,
    })
```

### 3. 문의 작성 (`/qna/write/`)

#### 핵심 기능
- **문의 작성 폼**: 제목, 내용 입력
- **공개/비공개 설정**: PUBLIC 또는 PRIVATE 선택
- **개인정보 수집 동의**: 필수 동의 체크
- **작성자 정보 자동 입력**: 로그인한 사용자 정보 자동 표시
- **개인정보 마스킹**: 전화번호, 이메일 마스킹 처리

#### 기술적 특징
```python
# 전화번호 마스킹 (010 6*** ****)
phone_parts = {
    'area': '010',
    'middle': '6***',  # 첫 글자만 표시
    'last': '****'     # 전체 마스킹
}

# 이메일 분리 (user@domain.com)
email_parts = {
    'username': 'user',
    'domain': 'domain.com'
}
```

#### 유효성 검사
- 제목, 내용 필수 입력
- 개인정보 수집 동의 필수

### 4. 문의 상세 조회 (`/qna/detail/`)

#### 핵심 기능
- **POST 방식 접근**: URL에 qna_id 노출 방지
- **권한 확인**: 본인 문의 또는 공개 문의만 조회 가능
- **답변 표시**: 관리자가 작성한 답변 내용 표시
- **작성자 정보**: 마스킹된 작성자 정보 표시

#### 보안 특징
```python
# POST 방식만 허용 (URL 노출 방지)
if request.method != 'POST':
    return redirect('qna:qna_list')

# 권한 확인
qna = get_object_or_404(
    Qna.objects.select_related('user').filter(
        Q(qna_id=qna_id) & (
            Q(user=user) |  # 본인 문의
            (Q(privacy='PUBLIC') & ~Q(user=user))  # 공개 문의
        )
    )
)
```

### 5. 문의 삭제 (`/qna/<qna_id>/delete/`)

#### 핵심 기능
- **본인 문의만 삭제**: 본인이 작성한 문의만 삭제 가능
- **더미 데이터 보호**: 더미 데이터는 삭제 불가
- **POST 방식만 허용**: 보안 강화

## 💡 기술적 도전과 해결

### 1. 상태별 우선 정렬
- **문제**: 답변 대기 문의를 먼저 표시해야 함
- **해결**: Case/When을 사용하여 상태값을 계산하고 정렬

### 2. AJAX 요청 처리
- **문제**: 페이지 전환 없이 검색/정렬 결과 업데이트
- **해결**: AJAX 요청 감지 후 부분 템플릿만 반환

### 3. 보안 강화
- **문제**: URL에 qna_id 노출 방지
- **해결**: POST 방식으로 상세 조회 처리

### 4. 자동 데이터 정리
- **문제**: 오래된 문의 데이터 관리
- **해결**: 30일 이전 데이터 자동 삭제 로직 구현

## 📝 결론

CareBridge의 1:1 문의 시스템은 관리자와 사용자 간의 효율적인 소통을 위한 핵심 기능입니다. 

**주요 특징**:
- ✅ 관리자 패널에서 통합 관리
- ✅ 사용자 친화적인 문의 작성 인터페이스
- ✅ 보안 강화 (접근 제어, 개인정보 보호)
- ✅ 성능 최적화 (쿼리 최적화, 페이지네이션)
- ✅ 자동 데이터 관리 (30일 이전 데이터 삭제)

이를 통해 사용자는 편리하게 문의를 작성하고, 관리자는 효율적으로 문의를 관리하고 답변할 수 있습니다.
