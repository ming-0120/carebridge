# 사용자 목록 관리창 기능 흐름도

## 개요
관리자 패널의 사용자 목록 관리 기능은 일반 사용자(환자)의 목록을 조회, 검색, 정렬, 페이지네이션하고, 선택한 사용자의 상세 정보를 확인할 수 있는 기능입니다.

---

## 전체 기능 흐름도

```
┌─────────────────────────────────────────────────────────────────┐
│                    사용자 목록 관리 페이지 진입                    │
│                    URL: /admin_panel/user_list/                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    views.py - user_list() 함수                   │
│                                                                   │
│  1. AJAX 요청 확인 (X-Requested-With 헤더)                        │
│  2. 파라미터 추출:                                                │
│     - search_type (검색 조건)                                     │
│     - search_keyword (검색어)                                     │
│     - user_id (선택된 사용자 ID)                                 │
│     - sort (정렬 필드)                                            │
│     - order (정렬 방향: asc/desc)                                 │
│     - page (페이지 번호)                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    데이터베이스 쿼리 처리                         │
│                                                                   │
│  1. 기본 쿼리셋:                                                  │
│     Users.objects.filter(role='PATIENT', withdrawal='0')          │
│                                                                   │
│  2. 정렬 적용:                                                    │
│     - 기본값: created_at desc (최신순)                           │
│     - 정렬 가능 필드:                                             │
│       • user_id, username, name, email, phone                    │
│       • gender, resident_reg_no, address, created_at            │
│                                                                   │
│  3. 검색 필터 적용:                                                │
│     - username__icontains (ID 검색)                              │
│     - name__icontains (이름 검색)                                │
│     - email__icontains (이메일 검색)                              │
│     - phone__icontains (전화번호 검색)                            │
│                                                                   │
│  4. 페이지네이션:                                                 │
│     - per_page=5 (페이지당 5개 항목)                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    데이터 가공 및 변환                             │
│                                                                   │
│  각 사용자 항목에 대해:                                           │
│  1. 번호 계산 (페이지네이션 고려)                                 │
│  2. 주민번호 → 생년월일 변환 (yyyy년mm월dd일)                    │
│  3. 신규 가입 유저 확인 (오늘 가입한 사용자)                      │
│  4. 주소 처리 (동까지만 표시)                                    │
│                                                                   │
│  선택된 사용자가 있으면:                                          │
│  1. 선택된 사용자 정보 조회                                       │
│  2. 즐겨찾기 병원 목록 조회                                        │
│  3. 주민번호 → 생년월일 변환                                      │

                              │
                              ▼
                    ┌─────────┴─────────┐
                    │                   │
                    ▼                   ▼

        │   AJAX 요청인가?   │  │   일반 요청인가?   │
        │  (user_id 있음)   │  │  (전체 페이지)    │
        └───────────────────┘  └───────────────────┘
                    │                   │
                    ▼                   ▼
        ┌───────────────────┐  ┌───────────────────┐
        │  JSON 응답 반환     │  │  HTML 페이지 렌더링│
        │  (상세 정보만)      │  │  (전체 페이지)    │
        │                     │  │                    │
        │  {                  │  │  user_list.html   │
        │    "detail_html":   │  │  템플릿 렌더링     │
        │    "<div>...</div>" │  │                    │
        │  }                  │  │  - 탭 메뉴         │
        └───────────────────┘  │  - 검색 섹션        │
                               │  - 사용자 목록 테이블│
                               │  - 페이지네이션      │
                               │  - 상세 정보 섹션    │
                               └───────────────────┘
```

## 데이터 흐름

### 서버 → 클라이언트 (응답)

```
┌─────────────────────────────────────────────────────────────────┐
│                    서버 응답 데이터 구조                          │
└─────────────────────────────────────────────────────────────────┘

1. 일반 페이지 요청 (GET):
   ┌─────────────────────────────────────────────────────────────┐
   │  HTML 응답 (user_list.html)                                 │
   │                                                              │
   │  컨텍스트 변수:                                              │
   │  - page_obj: 페이지네이션 객체                               │
   │  - users_with_number: 사용자 목록 (번호 포함)               │
   │  - search_type: 검색 조건                                    │
   │  - search_keyword: 검색어                                   │
   │  - selected_user: 선택된 사용자 (있으면)                   │
   │  - favorite_hospitals: 즐겨찾기 병원 목록                   │
   │  - birth_date: 생년월일                                     │
   │  - address_formatted: 포맷팅된 주소                         │
   │  - total_users_count: 전체 사용자 수                        │
   │  - search_result_count: 검색 결과 수                        │
   │  - sort_field: 정렬 필드                                    │
   │  - sort_order: 정렬 방향                                    │
   └─────────────────────────────────────────────────────────────┘

2. AJAX 상세 정보 요청 (POST with user_id):
   ┌─────────────────────────────────────────────────────────────┐
   │  JSON 응답                                                   │
   │  {                                                           │
   │    "detail_html": "<div class='user-detail-section'>...</div>"│
   │  }                                                           │
   │                                                              │
   │  detail_html 내용:                                           │
   │  - user_list_detail.html 템플릿 렌더링 결과                  │
   │  - 선택된 사용자의 상세 정보                                 │
   │  - 마스킹 처리된 개인정보 (주민번호, 전화번호)                │
   └─────────────────────────────────────────────────────────────┘
```

### 클라이언트 → 서버 (요청)

```
┌─────────────────────────────────────────────────────────────────┐
│                    클라이언트 요청 파라미터                        │
└─────────────────────────────────────────────────────────────────┘

1. 검색 요청 (POST):
   - search_type: 'username' | 'name' | 'email' | 'phone'
   - search_keyword: 검색어 (문자열)
   - csrfmiddlewaretoken: CSRF 토큰

2. 정렬 요청 (POST):
   - sort: 정렬 필드 이름
   - order: 'asc' | 'desc'
   - search_type: 검색 조건 (유지)
   - search_keyword: 검색어 (유지)
   - csrfmiddlewaretoken: CSRF 토큰

3. 페이지네이션 요청 (POST):
   - page: 페이지 번호
   - search_type: 검색 조건 (유지)
   - search_keyword: 검색어 (유지)
   - sort: 정렬 필드 (유지)
   - order: 정렬 방향 (유지)
   - csrfmiddlewaretoken: CSRF 토큰

4. 상세 정보 요청 (POST with AJAX):
   - user_id: 선택된 사용자 ID
   - csrfmiddlewaretoken: CSRF 토큰
   - Headers: X-Requested-With: XMLHttpRequest
```

## 주요 특징

### 1. AJAX 기반 비동기 처리
- 페이지 새로고침 없이 목록 업데이트
- 검색, 정렬, 페이지네이션 모두 AJAX 처리
- 상세 정보는 별도 AJAX 요청으로 로드

### 2. 개인정보 보호
- 주민번호 마스킹: 앞 6자리 + "-" + 7번째 자리 + "******"
- 전화번호 마스킹: 010 + 공백 + 첫 번째 숫자 + "****" + 공백 + "****"
- 목록에서는 주소를 동까지만 표시

### 3. 사용자 경험 (UX) 개선
- 스크롤 위치 유지 (페이지네이션 후)
- 선택된 행 시각적 표시 (selected 클래스)
- URL 파라미터로 상태 유지 (북마크/공유 가능)
- 검색조건 변경 시 검색어 자동 초기화

### 4. 성능 최적화
- 페이지네이션 (5개씩 표시)
- DB 레벨 정렬 (Python sorted() 사용 금지)
- 필요한 부분만 DOM 업데이트

### 5. 검색 기능
- 검색 조건: ID, 이름, 이메일, 전화번호
- 대소문자 구분 없음 (icontains)
- 전화번호 형식 검증

### 6. 정렬 기능
- 정렬 가능 필드: No., ID, 이름, 생년월일, 성별, 전화번호, 이메일, 주소, 생성일시
- 같은 필드 재클릭 시 정렬 방향 토글 (asc ↔ desc)
- 다른 필드 클릭 시 오름차순으로 시작
- 정렬 시 첫 페이지로 이동
