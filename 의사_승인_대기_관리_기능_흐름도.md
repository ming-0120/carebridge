# 의사 승인 대기 관리 기능 흐름도

## 개요
관리자 패널에서 의사 승인 대기 목록을 조회하고, 면허번호를 검증한 후 승인 또는 거절 처리를 수행하는 기능입니다.

## 주요 기능
1. 승인 대기 목록 조회 및 표시
2. 면허번호 자동 검증 (면허번호 뒷자리와 주민번호 뒷자리 비교)
3. 의사 선택 및 상세 정보 표시
4. 체크박스 선택 (개별/전체)
5. 승인 처리 (일괄 승인)
6. 거절 처리 (일괄 거절, 데이터 삭제)
7. 정렬 기능
8. 페이지네이션

## 주요 함수 및 역할

### 서버 사이드 (views.py)

#### `approval_pending(request)`
- **역할**: 승인 대기 목록 조회, 승인/거절 처리
- **GET 요청**:
  - 승인 대기 의사 조회 (`verified=False`)
  - 면허번호 검증
  - 정렬 처리
  - 페이지네이션 처리
  - 템플릿 렌더링
- **POST 요청**:
  - `action='approve'`: 선택된 의사 `verified=True`로 업데이트
  - `action='reject'`: 선택된 의사 및 관련 Users 삭제
  - AJAX 요청이 아닌 경우에만 리다이렉트

### 클라이언트 사이드 (approval_pending.js)

#### `selectDoctor(event, doctorId)`
- **역할**: 의사 선택 및 상세 정보 표시
- **동작**: `admin_common.js`의 `selectItem()` 함수 호출
- **결과**: AJAX로 상세 정보 로드 및 표시

#### `toggleSelectAllApproval()`
- **역할**: 전체 선택/해제
- **특징**: 면허번호 유효한 의사만 선택됨
- **동작**: 모든 개별 체크박스 상태 동기화

#### `updateSelectedDoctors()`
- **역할**: 선택된 의사 ID 수집 및 저장
- **동작**: 선택된 체크박스의 value 값을 수집하여 hidden input에 저장

#### `approveSelected()`
- **역할**: 선택된 의사 승인 처리
- **동작**: 확인 대화상자 → 폼 제출 (`action='approve'`)

#### `rejectSelected()`
- **역할**: 선택된 의사 거절 처리
- **동작**: 확인 대화상자 (되돌릴 수 없음 경고) → 폼 제출 (`action='reject'`)

#### `attachCheckboxListeners()`
- **역할**: 체크박스 이벤트 리스너 연결
- **동작**: 전체 선택 및 개별 체크박스에 이벤트 리스너 연결

#### `attachButtonListeners()`
- **역할**: 승인/거절 버튼 이벤트 리스너 연결
- **동작**: 승인 버튼과 거절 버튼에 클릭 이벤트 리스너 연결

## 데이터베이스 모델

### Doctors 모델
- `doctor_id`: 의사 고유 ID (Primary Key)
- `license_no`: 면허번호 (형식: "영어코드$주민번호7자리")
- `verified`: 검증 여부 (Boolean, False=승인 대기)
- `user`: Users 모델과 OneToOne 관계
- `hos`: Hospital 모델과 ForeignKey 관계
- `dep`: Department 모델과 ForeignKey 관계

### Users 모델 (관련 필드)
- `user_id`: 사용자 고유 ID
- `name`: 이름
- `username`: 로그인 ID
- `resident_reg_no`: 주민번호
- `phone`: 전화번호
- `created_at`: 계정 생성일시

## URL 패턴

```
/admin_panel/approval_pending/
- GET: 승인 대기 목록 조회
- POST: 승인/거절 처리
  - action: 'approve' 또는 'reject'
  - doctor_ids: '1,2,3' (쉼표로 구분된 의사 ID 목록)
```

## 주요 특징

1. **면허번호 자동 검증**
   - 면허번호 뒷자리와 주민번호 뒷자리를 자동으로 비교
   - 유효하지 않은 면허번호를 가진 의사를 우선 표시 (빨간색)

2. **일괄 처리 지원**
   - 체크박스로 여러 의사를 선택하여 한 번에 승인/거절 가능
   - 전체 선택 시 면허번호 유효한 의사만 선택됨

3. **AJAX 처리**
   - 행 클릭 시 상세 정보는 AJAX로 로드 (페이지 새로고침 없음)
   - 페이지네이션도 AJAX로 처리 (목록만 업데이트)

4. **안전장치**
   - 승인/거절 전 확인 대화상자 표시
   - 거절 시 "되돌릴 수 없습니다" 경고 메시지

5. **정렬 및 페이지네이션**
   - 다양한 필드로 정렬 가능
   - 페이지당 5개씩 표시
   - 정렬 상태 유지
