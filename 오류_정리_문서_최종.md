### 🔴 2025-12-02: 목록 페이지 개발 단계

#### 오류 #2: 페이지네이션 후 Python 정렬 함수 사용 문제
**발생 날짜**: 2025년 12월 2일  
**수정 날짜**: 2025년 12월 2일 ~ 12월 5일

**문제 상황**:
- **증상**: 페이지네이션 후 Python `sorted()` 함수를 사용하여 정렬하면 페이지 내부만 재정렬되어 전체 정렬이 깨짐
- **발생 위치**: `apps/admin_panel/views.py` - 모든 목록 페이지 (사용자목록, 의사목록, 병원목록, 승인대기, 1:1문의 등)

**잘못된 코드 예시**:
```python
# ❌ 잘못된 예시: 페이지네이션 후 정렬
page_obj, total_count = paginate_queryset(request, users, per_page=5)
users_with_number = sorted(
    users_with_number,
    key=lambda x: (not x['is_new_user'], x['user'].created_at),
    reverse=True
)  # 이렇게 하면 페이지 내부만 정렬되어 전체 정렬이 깨짐
```

**해결 방법**:
```python
# ✅ 올바른 예시: DB 레벨에서 정렬 후 페이지네이션
users = users.order_by('-created_at')  # DB에서 정렬
page_obj, total_count = paginate_queryset(request, users, per_page=5)
# 페이지네이션 후에는 정렬하지 않고 번호만 계산
users_with_number = []
for idx, user in enumerate(page_obj):
    users_with_number.append({
        'user': user,
        'number': start_index + idx
    })
```

**핵심 원칙**:
- **정렬은 반드시 DB 레벨에서 처리** (`order_by()` 사용)
- 페이지네이션 후에는 정렬하지 않고 번호만 계산
- 신규 가입자 표시 등 특별한 표시가 필요한 경우, 정렬 대신 템플릿에서 표시만 처리

**적용 범위**:
- 사용자목록 (`user_list`)
- 의사목록 (`doctor_list`)
- 병원목록 (`hospital_list`)
- 승인대기 (`approval_pending`)
- 1:1문의 (`qna_list`)

**해결 방법**:
```python
# ✅ 올바른 예시: AJAX 요청과 일반 요청 구분
```

#### 오류 #4: 이벤트 리스너 중복 연결 문제
**발생 날짜**: 2025년 12월 9일  
**수정 날짜**: 2025년 12월 9일 ~ 12월 10일

**문제 상황**:
- **증상**: DOM 업데이트 후 이벤트 리스너를 다시 연결할 때 중복 연결되어 이벤트가 여러 번 발생함
- **발생 위치**: 모든 JavaScript 파일 (예: `approval_pending.js`, `qna_list.js` 등)

**해결 방법**:
```javascript
// ✅ 올바른 예시: 기존 이벤트 리스너 제거 후 새로 추가
const checkbox = document.getElementById('selectAll');

// 기존 이벤트 리스너 제거
if (checkbox._changeHandler) {
    checkbox.removeEventListener('change', checkbox._changeHandler);
}

// 새로운 이벤트 핸들러 생성 및 저장
checkbox._changeHandler = function() {
    // 처리 로직...
};

// 이벤트 리스너 추가
checkbox.addEventListener('change', checkbox._changeHandler);
```

**핵심 원칙**:
- **기존 이벤트 리스너를 제거한 후 새로 추가**
- 이벤트 핸들러를 요소에 저장 (`_changeHandler`, `_clickHandler` 등)하여 나중에 제거 가능하도록 함
- DOM 업데이트 후 이벤트 리스너를 다시 연결할 때 반드시 적용

**적용 위치**:
- 체크박스 이벤트 리스너 (`attachCheckboxListeners`)
- 버튼 이벤트 리스너 (`attachButtonListeners`)
- 정렬 링크 이벤트 리스너 (`attachSortListeners`)

- **발생 위치**: `apps/admin_panel/templates/admin_panel/hospital_list.html` - 목록 템플릿

**가능한 원인**:

1. **AJAX 페이지네이션 시 `hospital_id` 파라미터가 포함되어 상세 정보만 반환되는 경우**
   - 페이지네이션 AJAX 요청 시 URL에 `hospital_id` 파라미터가 포함되어 있으면 서버가 상세 정보만 반환하려고 시도
   - 하지만 목록 템플릿이 렌더링되지 않아 데이터가 손실됨

2. **데이터베이스에 NULL 값이 저장되어 있는 경우**
   - 병원 모델에서 `tel`과 `estb_date` 필드가 `null=True, blank=True`로 설정되어 있어 NULL 값이 저장될 수 있음
   - 템플릿에서 `|default:"-"` 필터를 사용하지만, NULL 값이 제대로 처리되지 않을 수 있음

3. **템플릿에서 데이터 접근 방식 문제**
   - `item.hospital.tel`은 직접 접근하는데, `item.estb_date_formatted`는 딕셔너리 키로 접근
   - 일관성이 없어서 문제가 발생할 수 있음

**해결 방법**:
```html
<!-- 템플릿에서 더 명확한 NULL 체크 -->
<td>{% if item.hospital.tel %}{{ item.hospital.tel }}{% else %}-{% endif %}</td>
<td>{% if item.estb_date_formatted %}{{ item.estb_date_formatted }}{% else %}-{% endif %}</td>
```

**핵심 원칙**:
- 페이지네이션 AJAX 요청 시 `hospital_id` 파라미터 제거
- 템플릿에서 NULL 값 처리를 더 명확하게 수정
- 데이터베이스에 NULL 값이 있는지 확인
